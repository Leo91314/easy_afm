# 串口监视器怎么用

## 这一页是干什么的
教你把串口监视器当成“固件听诊器”：看状态、发命令、定位问题。

## 你会学到什么
- 串口监视器怎么开
- 本项目命令基本格式
- 如何做最小通信验证

- 30~50 分钟

## 正文

### 需要准备什么
- 已烧录的固件
- 正确串口端口
- 波特率 `115200`

### 一步一步怎么做
1. 打开串口监视器（PlatformIO Monitor 或其他串口工具）
2. 设置波特率为 `115200`
3. 发送一条 JSON 命令（每条命令一行）

示例命令：
```json
{"command":"get_status"}
```

如果要设置 DAC（示例）：
```json
{"command":"set_dac","channel":"F","value":32768}
```

如果要请求恢复默认状态：
```json
{"command":"restore"}
```

### 每一步完成后怎么检查
- 有合法 JSON 回复（例如 `status: success`）
- 错误命令能得到错误提示而不是死机

### 出错时先看哪里
- 波特率是否一致
- 命令是否是合法 JSON
- 是否忘了换行发送

### 暂时做不到也没关系的部分
- 暂时不必手动测试所有复杂命令（scan/approach 可放后面）

## 用最简单的话再说一遍
串口监视器是你和固件直接对话的地方，先用 `get_status` 打通就很关键。

## 在 red-panda-afm 项目里它对应什么
- 固件命令处理：`processCommand` 中 `get_status/set_dac/restore/...`

## 这一页完成后，你应该能做到什么
- 用串口验证固件是否在线
- 发送最小命令并读回响应

## 常见误区
- 用错波特率看成乱码就放弃
- 命令 JSON 格式错误（引号、逗号）
