# 先读 README 和 BUILD_GUIDE

## 这一页是干什么的
教你用“提取关键信息”的方式读项目文档，不是只浏览一遍。

## 你会学到什么
- README 和 BUILD_GUIDE 各负责什么
- 当前文档里明确写了什么
- 当前文档没写清什么

## 先决条件
- [[01-项目总览/03-red-panda-afm 项目结构总览]]

## 预计耗时
- 20~30 分钟

## 正文
当前仓库文档文件：
- `red-panda-afm/README.md`
- `red-panda-afm/README_zh.md`
- `red-panda-afm/BUILD_GUIDE.md`
- `red-panda-afm/BUILD_GUIDE_zh.md`

文档明确给出的信息（可确认）：
- 项目目标：低成本 DIY AFM
- 方向：3D 打印 + 电子 + 控制软件
- 关键对象：OPU、AFM 探针、压电执行器、微控制器
- 提醒了安全（激光、上电）

文档不足（需补充确认）：
- 未给出完整、可直接下单的 BOM
- 未给出独立发布的 Gerber/PnP
- 装配步骤偏概述，缺少精细图文和公差说明
- 性能指标写在 BUILD_GUIDE，但缺少测试条件与可复现实验方法

> [!warning] 现实提醒
> BUILD_GUIDE 更像“方向说明”，不是“工业级生产文件”。

## 用最简单的话再说一遍
文档能告诉你“要做什么”，但还不能让你“闭眼照做”。

## 在 red-panda-afm 项目里它对应什么
- 文档路径：`README*`、`BUILD_GUIDE*`

## 这一页完成后，你应该能做到什么
- 用“已确认/待确认”两栏做笔记
- 不再把 README 当成全量工程文档

## 常见误区
- 只看 README 不看源码和工程文件
- 把概述当成参数手册
