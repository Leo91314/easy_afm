# 采购记录模板

## 这一页是干什么的
提供标准采购记录表，支持追溯。

## 你会学到什么
- 如何记录采购状态
- 如何避免重复购买

## 先决条件
- [[05-采购与预算/01-采购总说明]]

## 预计耗时
- 建表 15 分钟，后续每次更新 3 分钟

## 正文
复制下面模板：

```markdown
# 采购记录

| 日期 | 类别 | 名称 | 型号/规格 | 数量 | 状态 | 证据来源 | 备注 |
|---|---|---|---|---|---|---|---|
| YYYY-MM-DD | | | | | 待确认/已确认/已下单/已到货 | 页面或文件路径 | |
```

## 用最简单的话再说一遍
买之前先记，买之后更要记。

## 在 red-panda-afm 项目里它对应什么
- 连接待确认页与实际下单动作

## 这一页完成后，你应该能做到什么
- 保持采购信息可追踪

## 常见误区
- 买了才记录，来源已找不到
