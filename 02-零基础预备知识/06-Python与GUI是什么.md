## 正文

### 1. Python 在这里不是“刷题语言”

在 `red-panda-afm` 中，Python 主要做上位机控制：
- 管理串口连接
- 发送控制命令
- 接收并处理数据
- 驱动图形界面显示

### 2. GUI 为什么有用

如果没有 GUI，你可能要一直手工敲命令。
GUI 的作用是：
- 把常用操作做成按钮/输入框
- 让你更直观看状态和曲线
- 降低操作门槛

### 3. 仓库里的确认事实

从源码 import 可以确认 GUI 依赖：
- PyQt6
- pyqtgraph
- pyserial
- numpy
- tifffile

主要文件：
- `gui/afm.py`：通信与逻辑层
- `gui/afm_gui.py`：界面层

### 4. 新手容易混淆的点

- GUI 能打开：只代表软件层基本可运行
- 要真正跑通：还需要固件、串口、硬件链路协同

## 用最简单的话再说一遍
Python+GUI 是“控制台和显示器”，不是硬件本体。
