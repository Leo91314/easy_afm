# 连接器针脚与网络对照表

## 这一页是干什么的
把“网络名级别信息”升级成“可接线可验收”的对照表模板，降低接线错误和方向错误风险。

## 你会学到什么
- 本项目关键连接器类型与网络族
- 从网络名到针脚号的落地步骤
- 对照表如何和实物连通测试联动

- 45~90 分钟

## 正文
### 重要提醒
本页默认你先有网络名，再补针脚号。没有 Pin1 方向验证，禁止上电。

### 关键网络族（首批必须覆盖）
- OPU 侧：`ADC_CS0/1` `DAC_CS0/1` `SCLK` `SDI` `SDO` `GND`
- Piezo 侧：`PIEZO_CS0/1/2` `PIEZO_SCLK` `PIEZO_SDI` `PIEZO_SDO`
- 步进/控制侧：`DIR` `STEP1/2/3` `SLEEP`
- 供电侧：`+5V` / 模拟电源 / 地

### 可直接使用的对照模板
| 连接器名 | 板名 | Pin号 | 网络名 | 方向(Pin1) | 实测连通 | 备注 |
|---|---|---|---|---|---|---|
| OPUData | PowerAndControl |  |  |  |  |  |

| PiezoData | PowerAndControl |  |  |  |  |  |

### 一步一步怎么做
1. 先在 epro 填“连接器名+网络名”。
2. 再补针脚号与 Pin1 方向。
3. 用万用表做连通测试，把结果填“实测连通”。
4. 发现冲突时，以“实测+原理图一致”为准修表。

### 每一步完成后怎么检查
- 每行都同时有：网络名 + Pin号 + 方向。
- 至少抽测 20% 关键线完成实测记录。

### 出错时先看哪里
- 网络名对不上：检查跨页网络标签。
- 连通测试不一致：优先排查连接器方向和焊接质量。

### 暂时做不到也没关系的部分
- 没有线束时可先做逻辑表，但不得直接上电联调。

## 用最简单的话再说一遍
先把“线该去哪”写成表，再把“线真到了哪”测出来。
