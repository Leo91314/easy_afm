# 如何打开gui项目

## 这一页是干什么的
把 GUI 从“能启动窗口”升级到“能连串口、能发命令、能看实时曲线”。

## 你会学到什么
- GUI 启动步骤与解释器绑定
- 主窗口、Focus、Approach、Scan 三个子流程怎么配合
- 首次启动时必须检查的状态项

- [[06-软件环境搭建/05-安装PyQt6等依赖]]
- [[06-软件环境搭建/08-如何打开firmware项目]]

- 30~55 分钟

## 正文
### 原理先讲清
GUI 核心文件：
- `afm.py`：串口协议与状态缓存
- `afm_gui.py`：窗口逻辑与数据可视化
- `ui/*.py`：Qt Designer 生成界面

GUI 实际上是“协议客户端 + 状态可视化器”，不是单纯界面壳。

### 一步一步怎么做
1. 进入 GUI 目录并激活虚拟环境。
2. 启动 GUI：
   ```bash
   cd red-panda-afm/gui
   source .venv/bin/activate
   python afm_gui.py
   ```
3. 在主窗口点击刷新串口，选择目标设备并连接。
4. 点击 `Reset` / `Restore` 做基础状态归零。
5. 观察实时曲线与 DAC/ADC 数值是否刷新。

### 每一步完成后怎么检查
- 窗口无崩溃，控制区可操作。
- 连接后 `Connected` 状态亮起。
- 主窗口实时曲线开始更新。
- 打开 Focus/Approach/Scan 子窗口不报错。

### 出错时先看哪里
- `ModuleNotFoundError`：依赖或解释器错误。
- 连接失败：串口号错、波特率错、固件未运行。
- 曲线不动：`get_status` 没有成功返回。

### 暂时做不到也没关系的部分
- 没硬件时可以只测“窗口能开 + 无依赖错误”。

## 用最简单的话再说一遍
GUI 的验收标准不是“能打开”，而是“能连、能控、能看数据”。

## 在 red-panda-afm 项目里它对应什么
- `red-panda-afm/gui/afm.py`
- `red-panda-afm/gui/afm_gui.py`
- `red-panda-afm/gui/ui/*.py`

## 这一页完成后，你应该能做到什么
- 独立启动并连接 GUI
- 在界面中完成最小控制回路验证

## 常见误区
- 用错 Python 解释器启动 GUI
- 忽略后台串口读线程状态
