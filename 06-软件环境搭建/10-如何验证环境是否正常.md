# 如何验证环境是否正常

## 这一页是干什么的
给你一套“软件阶段硬验收清单”，确保你进入第7章前具备可复现实验能力。

## 你会学到什么
- 固件链路、GUI链路、串口链路的通过标准
- 一组可直接执行的验收命令
- 验收失败时的优先排查顺序

## 先决条件
- [[06-软件环境搭建/08-如何打开firmware项目]]
- [[06-软件环境搭建/09-如何打开gui项目]]

## 预计耗时
- 25~40 分钟

## 正文
### 验收清单（全部通过再进下一章）
- [ ] `firmware` 编译通过（`pio run` 成功）
- [ ] GUI 启动无 import 错误
- [ ] GUI 能连接串口并执行 `get_status`
- [ ] 返回状态含 `state`、`adc_0_avg`、`dac_x/y/z`
- [ ] 你知道默认监视波特率 `115200`
- [ ] 你会保存构建/运行日志

### 建议你直接执行的命令
```bash
# 固件编译
cd red-panda-afm/firmware
pio run

# GUI导入检查
cd ../gui
source .venv/bin/activate
python -c "import PyQt6,pyqtgraph,serial,numpy,tifffile; print('deps ok')"

# GUI启动
python afm_gui.py
```

### 通过判定标准
1. 固件 `SUCCESS` 且无关键警告阻断。
2. GUI 打开后能刷新串口列表。
3. 连接后主界面数据开始刷新。
4. 至少完成一次 `reset` 或 `restore` 命令验证。

### 失败时排查顺序（按这个顺序省时间）
1. 解释器与虚拟环境是否正确。
2. 串口是否被其他软件占用。
3. 固件是否真在跑、波特率是否一致。
4. JSON 命令格式是否正确（含换行）。

### 暂时做不到也没关系的部分
- 没有硬件时，串口联调可标记为“待硬件验证”；其余条目必须通过。

## 用最简单的话再说一遍
第6章的目标不是“装完软件”，而是“能稳定执行一次完整软件闭环”。

## 在 red-panda-afm 项目里它对应什么
- `red-panda-afm/firmware/platformio.ini`
- `red-panda-afm/firmware/src/main.cpp`
- `red-panda-afm/gui/afm.py`
- `red-panda-afm/gui/afm_gui.py`

## 这一页完成后，你应该能做到什么
- 有资格进入 PCB 与机械实操阶段
- 后续出现问题时知道是环境问题还是硬件问题

## 常见误区
- 编译成功就算通过，忽略 GUI 与串口验证
- 不留日志，导致失败无法复盘
