# 电脑环境准备

## 这一页是干什么的
把你电脑变成“可复现 AFM 工程链路”的工作站，不是只装几个软件。

## 你会学到什么
- 软件复现需要的最小硬件/系统条件
- 推荐目录结构与版本记录方法
- 开工前 10 分钟自检流程

## 先决条件
- [[04-复现总计划/04-第三阶段 完成软件环境]]

## 预计耗时
- 30~60 分钟

## 正文
### 原理先讲清
后续你会同时处理三类任务：
1. ESP32 固件编译与串口调试（PlatformIO + VSCode）
2. Python GUI 运行与依赖管理（PyQt6 + pyserial + pyqtgraph）
3. 文档/日志留痕（参数回溯）

所以电脑环境要同时满足 C++、Python、串口工具链，而不是单一开发环境。

### 需要准备什么
- 操作系统：Windows/macOS/Linux 皆可（建议 64 位）
- 可联网环境（首次安装依赖很重）
- 管理员权限（驱动、IDE 插件安装需要）
- 至少 10GB 可用空间（缓存 + 构建产物 + 日志）

### 推荐目录结构（按这个建，后面最省事）
```text
大学生物理实验竞赛/
├─ red-panda-afm/
├─ logs/
│  ├─ firmware-build/
│  ├─ gui-run/
│  └─ serial-capture/
└─ docs-notes/
```

### 一步一步怎么做
1. 在本机创建主目录 `大学生物理实验竞赛/`。
2. 确认 `red-panda-afm` 仓库已放在该目录下。
3. 进入仓库并做基线检查：
   ```bash
   cd red-panda-afm
   git status
   ```
4. 创建日志目录：
   ```bash
   mkdir -p ../logs/firmware-build ../logs/gui-run ../logs/serial-capture
   ```
5. 把“机器信息快照”写入一条记录（系统版本、CPU、内存、Python版本）。

### 每一步完成后怎么检查
- 你能在终端进入 `red-panda-afm`。
- `git status` 能正常输出当前分支与改动。
- 日志目录已创建，后续产物有固定归档位置。

### 出错时先看哪里
- `cd` 报路径不存在：先统一中英文空格与目录名。
- 权限不足：用管理员权限重开终端。
- 后续安装失败：先确认网络/代理策略。

### 暂时做不到也没关系的部分
- 没有硬件时，串口相关步骤可先跳过；但目录和日志体系必须先建好。

## 用最简单的话再说一遍
先把“工程地基”搭好，再装工具；否则后面每一步都会乱。

## 在 red-panda-afm 项目里它对应什么
- `red-panda-afm/firmware/`
- `red-panda-afm/gui/`
- `red-panda-afm/BUILD_GUIDE_zh.md`

## 这一页完成后，你应该能做到什么
- 明确后续软件链路依赖哪些环境条件
- 拥有可持续复盘的目录与日志结构

## 常见误区
- 工程放在下载目录、桌面多个副本混用
- 不记录系统基线，后期无法解释“为什么这台机能跑那台机不能跑”
