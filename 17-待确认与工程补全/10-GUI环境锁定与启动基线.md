- 最小依赖清单怎么整理

## 已知依赖线索（来自源码 import）
- `PyQt6`
- `pyqtgraph`
- `pyserial`
- `numpy`
- `tifffile`

> 说明：仓库当前没有完整 `requirements.txt`，所以需要你在本地补一个“可运行快照”。

## 建议锁定流程
1. 创建虚拟环境并安装依赖。
2. 运行 GUI 到主窗口可见。
3. 导出当前环境快照：`pip freeze > requirements.lock.txt`。
4. 记录 Python 主版本和启动命令。

## 启动基线检查清单
- [ ] `python --version` 已记录
- [ ] `pip list` 已保存
- [ ] GUI 可启动，无导入报错
- [ ] 无硬件时，程序能进入“未连接但可操作界面”状态

## 需要准备什么
- [[18-模板与记录/04-实验记录模板]]

## 一步一步怎么做
1. 在 `gui/` 目录创建并激活虚拟环境。
2. 安装源码所需依赖。
3. 启动 GUI 并截图保存。
4. 固化环境版本并归档到知识库。

## 每一步完成后怎么检查
- 是否可复现启动？
- 是否能在新终端按同样命令成功运行？

## 出错时先看哪里
- `ModuleNotFoundError`：查漏装依赖
- Qt 报错：确认 Python 与 PyQt6 版本兼容

## 暂时做不到也没关系的部分
- 先不追求多平台一致
- 先把你当前机器跑稳定

## 用最简单的话再说一遍
环境不锁定，今天能跑不代表明天能跑。

## 在 red-panda-afm 项目里它对应什么

- `red-panda-afm/gui/afm_gui.py`

## 常见误区
