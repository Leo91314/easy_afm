# 如何启动GUI

## 这一页是干什么的
给你一套可复制执行的 GUI 启动步骤，并告诉你启动后先看什么。

## 你会学到什么
- 启动前准备
- 启动命令
- 启动成功判据

## 先决条件
- [[06-软件环境搭建/05-安装PyQt6等依赖]]
- [[12-GUI与上位机部分/01-gui项目总览]]

## 预计耗时
- 15~25 分钟

## 正文

### 需要准备什么
- 已安装 Python
- 已创建并激活虚拟环境 `.venv`
- 已安装 `PyQt6 pyqtgraph pyserial numpy tifffile`

### 一步一步怎么做
1. 进入目录：
```bash
cd red-panda-afm/gui
```
2. 激活虚拟环境（macOS/Linux）：
```bash
source .venv/bin/activate
```
3. 启动 GUI：
```bash
python afm_gui.py
```

### 每一步完成后怎么检查
- 成功时：主窗口正常弹出，不秒退
- 若失败：终端会给出 ImportError 或其他异常栈

### 出错时先看哪里
- 虚拟环境是否激活
- 依赖是否缺失
- 是否在 `gui/` 目录运行

### 暂时做不到也没关系的部分
- 没连接硬件时，GUI 可以先只验证“启动与页面打开”

## 用最简单的话再说一遍
先让 GUI 稳定打开，再做连接和扫描。

## 在 red-panda-afm 项目里它对应什么
- 入口：`red-panda-afm/gui/afm_gui.py`

## 这一页完成后，你应该能做到什么
- 独立启动 GUI
- 知道启动失败优先排查哪里

## 常见误区
- 依赖装在系统 Python，不在项目虚拟环境里
- 运行路径不对
